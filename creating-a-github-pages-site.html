Создание сайта GitHub Pages

Вы можете создать сайт GitHub Pages в новом или существующем репозитории.

Создание репозитория для вашего сайта

Если ваш сайт является независимым проектом, вы можете создать новый репозиторий для хранения исходного кода вашего сайта. Если ваш сайт связан с существующим проектом, вы можете добавить исходный код в репозиторий этого проекта, в /docsпапку в ветке по умолчанию или в другой ветке. Например, если вы создаете сайт для публикации документации по проекту, который уже находится на GitHub, вы можете сохранить исходный код сайта в том же репозитории, что и проект.

Если учетная запись, которой принадлежит репозиторий, использует GitHub Free или GitHub Free для организаций, репозиторий должен быть общедоступным.

Если вы хотите создать сайт в существующем репозитории, перейдите к разделу « Создание своего сайта ».

1. В правом верхнем углу любой страницы используйте раскрывающееся меню и выберите Новый репозиторий (New repository).

2. Используйте раскрывающееся меню Владелец (Owner) и выберите учетную запись, которой вы хотите владеть репозиторием.

3. Введите имя репозитория и необязательное описание. Если вы создаете сайт пользователя или организации, ваш репозиторий должен называться <user> .github.io или <organization> .github.io. Если ваше имя пользователя или организации содержит прописные буквы, вы должны использовать строчные буквы. Дополнительные сведения см. в разделе « О страницах GitHub ».

4. Выберите видимость репозитория. Дополнительные сведения см. в разделе « О репозиториях ».

5. Выберите Инициализировать этот репозиторий с помощью файла README .

6. Щелкните Создать репозиторий (Create repository).

Создание вашего сайта

Прежде чем вы сможете создать свой сайт, у вас должен быть репозиторий для вашего сайта на GitHub. Если вы не создаете свой сайт в существующем репозитории, см. « Создание репозитория для вашего сайта ».

Предупреждение . Сайты GitHub Pages общедоступны в Интернете, даже если репозиторий сайта является частным. Если у вас есть конфиденциальные данные в репозитории вашего сайта, вы можете удалить их перед публикацией. Дополнительные сведения см. в разделе « О репозиториях ».

1. На GitHub перейдите в репозиторий вашего сайта.

2. Решите, какой источник публикации вы хотите использовать. Дополнительные сведения см. в разделе « О страницах GitHub ».

3. Если выбранный вами источник публикации уже существует, перейдите к источнику публикации. Если выбранный вами источник публикации не существует, создайте источник публикации.

4. В корне источника публикации создайте новый файл с именем index.md, который содержит контент, который вы хотите отобразить на главной странице вашего сайта.

Совет: если index.html присутствует, то будет использоваться вместо index.md. Если ни один index.html из index.md них не присутствует, README.md будет использоваться.

5. Настройте источник публикации. Дополнительные сведения см. в разделе « Настройка источника публикации для вашего сайта GitHub Pages ».

6. Под именем вашего репозитория нажмите Настройки .

7. В разделе «Код и автоматизация» боковой панели нажмите « Страницы » .

8. Чтобы просмотреть опубликованный сайт, в разделе «Страницы GitHub» щелкните URL-адрес своего сайта.

Примечание. Публикация изменений на вашем сайте может занять до 10 минут после отправки изменений на GitHub. Если через час вы не видите изменения своего сайта GitHub Pages в браузере, см. раздел « Об ошибках сборки Jekyll для сайтов GitHub Pages ».

9. Если ваш сайт GitHub Pages создается из общедоступного репозитория, он создается и развертывается с помощью запуска рабочего процесса GitHub Actions, если вы не настроили свой сайт GitHub Pages для использования другого инструмента CI. Дополнительные сведения о том, как просмотреть состояние рабочего процесса, см. в разделе « Просмотр истории выполнения рабочего процесса ».

Примечание. Рабочий процесс GitHub Actions для ваших сайтов GitHub Pages находится в общедоступной бета-версии для общедоступных репозиториев и может быть изменен. Запуски рабочих процессов GitHub Actions бесплатны для общедоступных репозиториев.

Примечание. Если ваш сайт не был опубликован автоматически, убедитесь, что кто-то с правами администратора и подтвержденным адресом электронной почты отправил его на источник публикации.

Следующие шаги

Вы можете добавить больше страниц на свой сайт, создав больше новых файлов. Каждый файл будет доступен на вашем сайте в той же структуре каталогов, что и источник публикации. Например, если источником публикации для вашего сайта проекта является gh-pages ветка, и вы создаете новый файл с именем /about/contact-us.md в gh-pages ветке, этот файл будет доступен по адресу https://<user> .github.io/<repository> /about/contact-us.html.

Вы также можете добавить тему, чтобы настроить внешний вид вашего сайта. Дополнительные сведения см. в разделе « Добавление темы на сайт GitHub Pages с помощью средства выбора темы ».

Чтобы еще больше настроить свой сайт, вы можете использовать Jekyll, генератор статических сайтов со встроенной поддержкой страниц GitHub. Дополнительные сведения см. в разделе « О GitHub Pages и Jekyll ».
